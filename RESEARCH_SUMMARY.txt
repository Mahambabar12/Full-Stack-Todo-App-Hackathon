KEY FINDINGS
================================================================================

DIALOG COMPONENT
- Best for: Modal interactions, forms, confirmations
- State: Use controlled with useState
- Form: Integrate with React Hook Form + Zod
- Best Practice: Reset form and close only after success
- Accessibility: Automatic via Radix UI (focus trap, keyboard nav)

POPOVER COMPONENT
- Best for: Inline selections, non-modal interactions
- State: Controlled via useState
- Auto-close: Set setOpen(false) after selection
- Pattern: Combine with Command for search capability
- UX: Lightweight, contextual alternatives

STATE MANAGEMENT PATTERNS
1. Uncontrolled Dialog - Simple, no parent coordination
2. Controlled Dialog - Recommended, parent manages state
3. Multiple Dialogs - Separate useState per dialog
4. Popover State - Auto-close after selection

ACCESSIBILITY FEATURES (Built-in)
- Focus automatically moves to dialog when opened
- Focus trap ensures user can't tab outside
- Escape key closes dialog automatically
- Screen reader support for title and description
- Full keyboard navigation (Tab, Shift+Tab, Escape)
- ARIA labels applied automatically

================================================================================
COMMON MISTAKES & SOLUTIONS
================================================================================

Mistake 1: Closing dialog before API responds
Solution: Close only after successful submission

Mistake 2: Not resetting form
Solution: Call form.reset() after successful submission

Mistake 3: Missing loading state
Solution: Add disabled={isSubmitting} to button

Mistake 4: Using uncontrolled state
Solution: Use useState to control from parent

Mistake 5: Dialog inside dropdown
Solution: Wrap Dialog outside DropdownMenu

Mistake 6: Missing title/description
Solution: Always include DialogTitle and DialogDescription

Mistake 7: Forgetting validation
Solution: Use Zod schemas with React Hook Form
